# Docs Index

本目录按“当前有效文档”与“历史归档文档”分层管理，避免顶层堆积审计报告。

## 当前有效（顶层）

- `ARCHITECTURE_REPORT.md`: 当前架构现状与问题分级（主报告）。
- `Gaokao_Constraint_Protocol_v1.0.md`: 项目约束协议与质量门槛。
- `data_processing_pipeline_design.md`: 数据处理管道设计方案。
- `user-service-refactoring-plan.md`: 用户服务重构计划（专项规划）。

## 历史归档

- `archive/audit-reports/`: 历史架构/前后端/数据/AI/GUI审计报告与阶段性复盘。
- `archive/fix-reports/`: 历史修复报告与交付说明。
- `archive/misc/`: 旧版指南、推进记录与杂项资料。

## 维护规则

- 新文档优先放到对应业务目录；仅“当前生效的总览文档”放在 `docs/` 顶层。
- 阶段性审计、临时分析、一次性报告在完成后移动到 `archive/`。
- 同主题出现新版后，旧版移入 `archive/`，避免并列重复。
